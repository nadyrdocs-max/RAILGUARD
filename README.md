# RAILGUARD

**RailGuard** is a research project focused on detecting railway wheel-flat defects using vibration analysis and machine learning.

The idea behind the project is simple: when a wheel develops a flat spot, every wheel revolution produces an impact. These impacts create characteristic vibration patterns that can be measured with an accelerometer mounted near the axle box. If these patterns can be detected reliably, maintenance crews may identify damaged wheels before they cause excessive wear, noise, or safety issues.

This repository contains the simulation environment developed during the early stage of the project. The simulation was created to study how wheel-flat size influences vibration signals and to determine which signal-processing features are most sensitive to defect severity.

---

## Research Question

Can wheel-flat defects be detected and classified using vibration signals measured from an axle-mounted accelerometer?

To investigate this question, the simulation studies the relationship between:

* wheel-flat size,
* wheel rotational speed,
* vibration response,
* extracted signal features.

## Simulation Approach

The current model represents a simplified wheel–roller system.

Instead of reproducing a complete railway vehicle, the simulation focuses on the mechanism that is most important for this research:

**periodic impacts generated by a wheel flat.**

The model includes:

* wheel geometry,
* wheel-flat defects,
* rotational motion,
* impact excitation,
* vibration response,
* measurement noise.

The objective is not to predict exact railway forces but to reproduce realistic trends in vibration behaviour as defect severity increases.

## Extracted Features

Several commonly used vibration-diagnostics features are calculated:

* RMS
* Peak Value
* Crest Factor
* Kurtosis
* Skewness
* Shape Factor
* Impulse Factor
* Spectral Entropy
* Band Energy (1000–3000 Hz)

These features are used to evaluate whether defect severity can be distinguished statistically.

## Results

The simulation shows several consistent trends:

* wheel flats generate periodic impacts;
* larger defects produce stronger vibration signatures;
* impulse-related features increase with defect size;
* frequency content changes as impact severity increases.

These results suggest that vibration-based wheel-flat detection is technically feasible and worth validating experimentally.

---

## Limitations

This is a research simulation and not a full railway dynamics model.

Several simplifications were made:

* reduced-scale geometry,
* simplified wheel–rail contact,
* synthetic measurement noise,
* simplified structural dynamics.

For this reason, absolute vibration amplitudes should not be interpreted as real railway measurements.

The simulation is intended to study **relative trends**, not exact railway behaviour.

## Hardware Validation

The next stage of RailGuard is the construction of a laboratory test rig.

Planned hardware includes:

* Raspberry Pi 5
* ADXL355 accelerometer
* Hall-effect encoder
* Load cell
* DC gear motor
* Interchangeable wheel sets
* Steel roller

The purpose of the hardware phase is to compare experimental measurements with simulation predictions and identify which features remain useful under real-world conditions.
